# Architecture Decision Records (ADR)

[English](../../../docs/adr/README.md)

## 什麼是 ADR？

架構決策紀錄 (ADR) 是一份文件，用於記錄重要的架構決策，以及該決策背後的背景脈絡和後續影響。

### 撰寫範本

新增 ADR 時，請使用以下範本 (檔名格式：`NNNN-決策標題-kebab-case.md`)：

```markdown
# NNNN. 決策標題

- **狀態**: [提議中 | 已接受 | 已取代 | 已廢棄]
- **日期**: YYYY-MM-DD
- **作者**: [姓名]

## 背景 (Context)
描述促使做出此決策的問題或機會。解釋相關的影響因素（技術、政策、團隊現況等）。

## 決策 (Decision)
說明所做出的決策內容。使用主動語氣（「我們將...」）。

## 後果 (Consequences)
描述執行此決策後的結果狀況。應包含所有影響，不僅是好的方面。
- **正面影響**: ...
- **負面影響**: ...
- **風險**: ...
```
