# Deprecation Policy

[English](../deprecation.md)

## 策略

我們的目標是為使用者維持穩定性。然而，為了平台演進，某些功能可能會被淘汰 (Deprecate)。

1. **通知期**: 我們承諾在移除公開 API 或功能前，提供至少 **6 個月** 的通知期。
2. **溝通方式**: 淘汰訊息將在發布說明 (Release Notes) 中公告，並於文件中標註。
3. **執行期警告**: 被淘汰的 API 將盡可能回傳 `Warning` HTTP 標頭。

### 淘汰生命週期

1. **草案 (Draft)**: 內部討論階段。
2. **已淘汰 (Deprecated)**: 文件中標註已淘汰。功能完全正常。加入 `Warning` 標頭。
3. **終止支援 (EoS)**: 功能可能仍可運作，但不提供錯誤修正。
4. **已移除 (Removed)**: 功能被完全移除。端點回傳 `404` 或 `410 Gone`。

### 目前淘汰項目

*目前無。*
